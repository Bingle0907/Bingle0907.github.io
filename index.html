UIStackView使用及剖析

发表于 2015-11-07   |   分类于 iOS
    

UIStackView 是iOS9新推出的一个UIKit视图，UIStackView 的核心在于能够非常方便地垂直或者水平排布多个视图，我们不用再拉很多的约束，只需要将几个视图丢进去，它会为其Subviews 自动添加约束，约束将不再由开发者管理。

UIStackView 会自动根据设备朝向、屏幕尺寸、及Subview的isHidden等来动态调整Subviews 的布局与尺寸，我们可以通过设定UIStackView 的属性：axis、distribution、alignment、spacing等来提供一个精确的布局。而且UIStackView 可以嵌套使用，在UIStackView 上添加UIStackView 来完成更加复杂的布局。

UIStackView Auto Layout

  • 对于StackView 的约束，通常情况下只需要为他确定position便可确定layout，长度与宽度会根据StackView 内部的arranged views的布局来自主拉伸，arranged views大部分情况下会保持原尺寸而不被压缩，部分情况下面会介绍。

  • 如果对StackView的长度、宽度做了约束后，StackView 将对arranged views进行拉伸或压缩来适应。

1.垂直Axis方向无约束宽度的情况下

StackView垂直Axis方向的长度一般情况下都是由Subviews中在该方向长度最长的决定。

只有在Alignment = fill情况下， arranged views中view的Content Compression Resistance Priority（压缩抵抗等级）值不一致时，StackView的该方向长度将由压缩抵抗等级最高中，长度最长的那个view决定。

2.Axis方向上无长度约束情况下：

该情况下在Distribution != Fill Equally 情况下，arranged views的高度均为原高度，StackView的长度由arranged views的原有高度加上Space的高度决定。

只有在Distribution == Fill Equally，每个subview的高度等于arranged views中，压缩抵抗等级最高中，高度最高的那个。如图当降低image view的压缩抵抗等级后，arranged views的高度均等于Label的高度。

  1. 为StackView添加长度宽度约束情况下：

当arranged views不足以填充StackView 长度时将会根据Content Hugging Priority（拉伸等级）来判断拉伸视图，会优先拉伸拉伸等级较低的视图。当Stackview 无法容纳arranged views时将根据压缩抵抗等级优先压缩等级较低的视图。如果arranged views的等级一样的情况下，很多时候StackView就会报错误，因为无法判断优先拉伸或压缩哪个视图来做布局。至于在这种情况下具体的Distribution决定的布局方式我们可以往下看。

UIStackView Property

  • Distribution负责Axis方向上layout。
    UIStackViewDistributionFill : stack view 会修改arranged views 来使得axis方向的布局得到填充。

UIStackViewDistributionFillEqually : stack view 会修改arranged views 尺寸使得每个的尺寸一致并填充。

UIStackViewDistributionFillProportionally : stack view 会修改arranged views的原尺寸大小按比例拉伸或压缩尺寸来填充。

UIStackViewDistributionEqualSpacing : stack view 调整arranged views 的position 来保证每个view的间隔一致，如果不够填充则拉伸相同间隔。

UIStackViewDistributionEqualCentering : stack view 调整arranged views 的position 使得每个view的中心间距一致。如果无法容下该布局则先缩小间距直到最小间距，还是无法容下时开始压缩视图大小。

  • Alignment负责的是垂直与Axis方向上的layout：

    从下面示意图我们可以很容易明白他们各自的作用。

  • UIStackView是UIView的子类，不具有user interface ，他的background Color 属性是无效的，同时也不能重写他的 layerClass: ;drawRect: 等方法；
  • spacing 属性确定arranged views之间在Axis方向的最小间距。
  • baselineRelativeArrangement 其视图间的垂直间隙是否根据基线测量得到.
  • layoutMarginsRelativeArrangement 决定了 stack 视图平铺其管理的视图时是否要参照它的布局边距.
